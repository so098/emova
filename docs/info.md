# Emova (감정 기반 실행 루틴 코칭 앱)

## 1. 한 줄 정의

감정을 출발점으로 사용자의 실행 루틴을 설계하고,
미션 → 실행 → 회고 → 성장 루프를 형성하는 웹앱

---

## 2. 핵심 문제 정의

기존 루틴 앱의 문제:

- 행동 추적 중심 → 감정 변화 고려 부족
- 목표 설정은 가능하지만 실행 지속이 어려움
- 감정 기록은 있어도 실행으로 연결되지 않음

➡ Emova 해결 방향:

- 감정을 “행동의 트리거”로 활용
- 감정 → 실행 → 회고까지 하나의 루프로 설계

---

## 3. 핵심 UX 플로우

1. 감정 선택 (또는 스킵)
2. 오늘의 미션 선택 또는 생성
3. 실행 전 메모 (선택)
4. 미션 완료 체크
5. 회고 작성 (감정 변화 + 텍스트)
6. 보상 지급 (XP / 포인트)
7. 감정-행동 기록 누적

---

## 4. 핵심 기능

### 4.1 감정 입력

- 선택형 감정 (불안, 무기력, 의욕 등)
- 텍스트 입력 가능

### 4.2 미션 구조

- 장기 미션 (목표)
- 단기 미션 (실행 단위)
- 하루 1회 루틴 중심 (MVP 기준)

### 4.3 실행 메모

- 실행 전 생각 / 다짐 기록
- 회고 시 자동 불러오기

### 4.4 회고 시스템

- 감정 변화 선택 (이전 → 이후)
- 텍스트 회고
- AI 피드백 (예정)

### 4.5 보상 시스템

- XP: 성장 경험
- 포인트: 외적 보상

---

## 5. 핵심 설계 철학

### 5.1 감정은 입력 데이터

감정은 단순 기록 대상이 아니라
행동을 유도하는 신호로 사용됨

### 5.2 실행 최소화 전략

작은 행동부터 시작하도록 설계
→ 자기효능감 회복 유도

(참고: BJ Fogg Behavior Model)

### 5.3 회고 = 성장 루프 완성

회고는 선택 기능이 아니라
“행동-감정 연결을 인식하는 핵심 단계”

---

## 6. 차별점

| 항목   | 기존 앱   | Emova            |
| ------ | --------- | ---------------- |
| 시작점 | 목표      | 감정             |
| 중심   | 행동 기록 | 감정-행동 연결   |
| 회고   | 선택적    | 핵심 루프        |
| 동기   | 외적      | 내적 + 외적 분리 |

## 7. 기술 스택 (예정)

- React
- Tailwind CSS
- Zustand or Redux
- Supabase (DB / Auth)
- GPT API (회고 피드백)

## 8. MVP 범위

포함:

- 감정 선택
- 미션 생성 및 완료
- 회고 작성
- 기본 보상

제외:

- 감정 히스토리 그래프
- AI 심화 분석
- 게임화 확장 기능

## 9. 타겟 사용자

- 감정은 있지만 실행으로 연결 못하는 사용자
- 기존 루틴 앱이 맞지 않는 사용자
- 자기이해 초기 단계의 20~30대
