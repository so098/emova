# 완성도 개선 계획

> 작성: 2026-04-08

MVP 기능은 대부분 완료. 아래는 체감 퀄리티를 높이기 위한 작업 우선순위.

---

## 1. 온보딩 (우선순위: 최상)

첫 진입 사용자가 뭘 해야 하는지 모른다는 피드백 있음.

- [ ] 첫 방문 시 2~3장 슬라이드 (앱 목적 + 핵심 흐름 설명)
- [ ] 코치마크 또는 툴팁으로 주요 UI 안내
- [ ] `localStorage`로 첫 방문 여부 판단, 한 번만 노출

---

## 2. 반응형 (우선순위: 상)

현재 레이아웃이 데스크톱 기준. 모바일 사용자가 대부분일 것.

- [ ] 전 페이지 모바일(375px~) 레이아웃 점검
- [ ] NavMenu 모바일 하단 고정 동작 확인
- [ ] 모달/오버레이 모바일에서 잘리는지 확인
- [ ] textarea, 버튼 터치 영역 충분한지 체크

---

## 3. 빈 상태(Empty State) 개선 (우선순위: 중상)

데이터 없을 때 빈 화면이 허전함.

- [ ] 퀘스트 없을 때: 안내 문구 + 생성 CTA
- [ ] 회고 없을 때: 안내 문구 + 회고 쓰기 CTA
- [ ] 프로필/업적 비어있을 때: 힌트 텍스트

---

## 4. 로딩/에러 상태 통일 (우선순위: 중)

퀘스트 페이지만 스켈레톤 로딩 있고 나머지는 없음.

- [ ] 회고, 프로필, 상점 페이지에 스켈레톤 추가
- [ ] 에러 시 인라인 안내 (토스트 or 에러 카드)
- [ ] 전체 로딩 경험 통일

---

## 5. 감정 선택지 보강 (우선순위: 중)

현재 부정 감정 위주 10개, 긍정은 "설렘" 1개뿐.

- [ ] 긍정 감정 추가: 뿌듯함, 편안함, 감사함, 기대감 등
- [ ] 회고 감정 선택(`ReflectPage`)에도 동일 적용
- [ ] 감정 카드(`EmotionCardList`)에도 반영

---

## 6. OAuth 로그인 (우선순위: 중)

현재 익명 인증 — 기기 바꾸면 데이터 소실.

- [ ] Google/Kakao OAuth 연동
- [ ] 인증 완료 전 쓰기 UI 비활성화 (레이스 컨디션 방지)
- [ ] 익명 → OAuth 계정 전환(merge) 지원 여부 결정
