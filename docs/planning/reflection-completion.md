# 회고 완주율 높이기 — 기획안

## 문제 정의

Emova의 핵심 루프는 **감정 → 미션 → 실행 → 회고 → 성장**이다.
이 중 회고는 루프를 완성하는 마지막 단계지만, 사용자 이탈이 가장 많이 발생하는 구간이기도 하다.
회고 없이는 감정-행동 패턴 데이터가 쌓이지 않고, 앱의 핵심 가치가 작동하지 않는다.

---

## 원칙

- **회고는 3초 안에 끝날 수 있어야 한다** — 최소 행동 단위를 극단적으로 낮춘다
- **안 하는 것보다 짧게라도 하는 게 낫다** — 텍스트 없이 감정 탭만 눌러도 완료 처리
- **회고의 가치는 누적에서 나온다** — 단일 회고는 가벼워도, 쌓이면 패턴이 보인다

---

## 전략

### 1. 마찰 최소화

| 항목 | 현재 | 개선안 |
|---|---|---|
| 최소 입력 | 감정 변화 선택 + 텍스트 | 감정 변화 탭 1회만으로 완료 |
| 텍스트 입력 | 필수 or 강조 | 선택사항 (보너스 XP 부여) |
| 화면 전환 | 별도 진입 필요 | 미션 완료 직후 자동 전환 |

**핵심:** 사용자가 해야 할 행동을 딱 하나로 줄인다.
감정 변화 선택 한 번이면 회고 완료. 텍스트는 보너스.

### 2. 타이밍 설계

- **미션 완료 → 회고 자동 연결**: 미션 완료 버튼을 누르면 바로 회고 화면으로 전환. "나중에 하기"는 있되, 기본 흐름은 즉시 회고.
- **미완료 회고 리마인더**: 미션은 완료했지만 회고를 안 한 경우, 다음 접속 시 "어제 미션 어땠어?" 카드 노출.
- **시간 지나면 간소화**: 24시간 이후의 회고는 감정 선택만으로 단축 (기억이 희미해진 텍스트 회고는 의미 없음).

### 3. 보상 구조

| 행동 | XP | 비고 |
|---|---|---|
| 미션 완료 | 기본 XP | 현행 유지 |
| 회고 완료 (감정만) | 미션 XP의 50% 추가 | 회고 인센티브 |
| 회고 완료 (텍스트 포함) | 미션 XP의 100% 추가 | 텍스트 작성 보너스 |
| 연속 회고 streak | streak일수 × 보너스 | 3일, 7일, 14일 구간 강조 |

**설계 의도:** 미션만 하고 회고를 안 하면 XP의 절반밖에 못 받는 구조.
회고가 "선택"이 아니라 "당연히 하는 것"이 되도록 유도.

### 4. 회고 잠금 (Lock) 옵션

- **Soft Lock**: 회고를 안 하면 다음 날 미션 목록에 "어제 회고 먼저" 배너 노출. 미션 생성은 가능하되 시각적으로 회고를 우선 유도.
- **Hard Lock** (선택적): 회고를 완료해야 다음 미션이 열림. 강제성이 높아 이탈 리스크 있음 → 설정에서 사용자가 선택하게 할 수도 있음.

> 초기에는 Soft Lock으로 시작하고, 데이터 보고 판단.

### 5. 패턴 피드백 (누적 가치)

회고가 쌓이면 감정-행동 연결 패턴을 보여줄 수 있다.
이것이 사용자가 "회고를 계속 해야 하는 이유"가 된다.

- **주간 요약**: "이번 주 불안할 때 산책 미션을 했고, 3번 중 2번 감정이 개선됐어"
- **패턴 알림**: "무기력할 때 운동 미션의 감정 개선율이 80%야"
- **트리거 제안**: "불안한 날에는 산책 미션이 잘 맞았어. 오늘도 해볼까?"

> 이 기능은 데이터가 충분히 쌓인 후 (최소 2주) 활성화.
> MVP에서는 단순 통계만 보여주고, AI 분석은 이후 단계.

## 구현 우선순위

| 순서 | 항목 | 난이도 | 임팩트 |
|---|---|---|---|
| 1 | 미션 완료 → 회고 자동 전환 | 낮음 | 높음 |
| 2 | 감정 탭만으로 회고 완료 처리 | 낮음 | 높음 |
| 3 | 회고 XP 보상 차등 적용 | 중간 | 높음 |
| 4 | 연속 회고 streak 보너스 | 중간 | 중간 |
| 5 | 미완료 회고 리마인더 카드 | 중간 | 중간 |
| 6 | Soft Lock 배너 | 낮음 | 중간 |
| 7 | 주간 패턴 요약 | 높음 | 높음 (장기) |

---

## 검증 지표

- **회고 완료율**: 미션 완료 대비 회고 완료 비율 (목표: 70% 이상)
- **텍스트 회고 비율**: 전체 회고 중 텍스트를 작성한 비율 (목표: 30% 이상)
- **회고 streak 분포**: 연속 회고 일수별 사용자 분포
- **이탈 지점**: 회고 화면 진입 후 이탈율, 어느 단계에서 이탈하는지
