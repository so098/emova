# Emova 프로젝트 현황 평가

> 작성일: 2026-04-03

## 전체 요약

MVP 프론트엔드의 핵심 플로우가 완성된 상태. 감정 선택 → 질문 응답 → AI 추천 → 퀘스트 관리까지 한 사이클이 돌아가고, UI/애니메이션 완성도가 높다.

---

## 잘 된 부분

### 구조 & 아키텍처

- 컴포넌트(`components/`) vs 도메인 기능(`features/`) 분리가 깔끔함
- 라우트 상수 중앙관리(`routes.ts`), 디자인 토큰 CSS 변수 체계가 잘 잡혀있음
- Server Component / Client Component 분리가 적절함 — 추천 페이지는 서버에서 Claude API 호출 후 결과를 클라이언트로 전달

### UX & 인터랙션

- Framer Motion 활용이 뛰어남 — 드래그 페이지네이션, 스프링 애니메이션, 축하 파티클 등
- 감정 카드 캐러셀, 퀘스트 인라인 편집, 추천 멀티셀렉트 등 인터랙션이 세밀함
- 모바일 반응형 대응 (NavMenu 하단 탭 전환 등)

### 코드 품질

- TypeScript strict 모드, 타입 안정성 높음
- Zustand 2개 스토어가 가볍고 명확한 역할 분담
- URL params 기반 상태 전달로 서버/클라이언트 경계 자연스러움

## 우려되는 부분

| 영역 | 현황 | 리스크 |
|---|---|---|
| **데이터 영속성** | 모든 상태가 메모리/URL — Supabase 패키지만 설치, 미연동 | 새로고침하면 퀘스트·진행상황 전부 소실 |
| **인증** | 없음 | 사용자 식별 불가, 데이터 저장 시 필수 선행 |
| **퀘스트 상태** | `QuestPage.tsx` 내부 하드코딩 목업 데이터 | 실제 CRUD 없이 UI만 존재 |
| **진행도 리셋** | `progressStore`에 reset 없음, 단방향 전진만 가능 | 플로우 재시작 불가 |
| **에러 처리** | `actions.ts`에서 Claude API 실패 시 기본적인 throw만 | 네트워크 오류, rate limit 등 사용자 피드백 부재 |
| **테스트** | 0개 | 리팩토링·기능 추가 시 회귀 버그 위험 |
| **접근성** | Skip link 존재하나 ARIA label 대부분 누락 | 스크린 리더 사용자 지원 부족 |

---

## MVP 완성까지 남은 핵심 갭

1. **Supabase 연동** — 사용자 인증 + 퀘스트/감정 기록 DB 저장 (현재 가장 큰 갭)
2. **회고 기능** — NavMenu에 "회고" 탭이 있지만 페이지 미구현
3. **XP/포인트 시스템** — UI는 있으나(`XPBar`, `EmotionGauge`) 실제 적립 로직 없음
4. **진행도 초기화** — 플로우 완료 후 다시 시작하는 메커니즘

---

## 판단

프론트엔드 프로토타입으로서는 완성도가 높은 편. 디자인 시스템, 애니메이션, 컴포넌트 구조 모두 MVP 출시 수준에 근접. 다만 백엔드 연동 없이는 데모 이상의 가치를 만들기 어려운 상태이므로, 다음 단계는 Supabase 인증 + 퀘스트 CRUD 영속화가 되어야 함.
