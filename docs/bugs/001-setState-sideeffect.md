# setState 업데이터 안에서 토스트 호출 → 중복 토스트

**날짜:** 2026-04-03

**증상:**
퀘스트 완료 시 동일한 토스트가 두 개씩 나옴.

**원인:**
`setQuests((prev) => { ... })` 업데이터 함수 안에서 `showToast()`를 호출함.
React 18 Strict Mode는 개발 모드에서 업데이터 함수를 **두 번 호출**하여 순수성을 검증함.
기존 `CelebrationToast`는 단일 visible 토글 방식이라 두 번 호출돼도 결과가 같아서 문제가 안 보였지만,
`ToastStack`(배열 push 방식)으로 교체하면서 두 번 호출 = 토스트 두 개가 됨.

**잘못된 코드:**
```ts
setQuests((prev) => {
  const next = { ...순수 변환 };
  showToast("완료!");  // ← 업데이터 안에서 사이드이펙트
  return next;
});
```

**수정:**
상태 업데이트(순수)와 사이드이펙트(토스트/모달)를 분리.

```ts
const toggled = [...quests.단기, ...quests.장기].find((q) => q.id === id);

setQuests((prev) => {
  return { ...순수 변환 };
});

showToast("완료!");
```

**교훈:**
- `setState` 업데이터 함수는 **순수 함수**여야 한다 (상태 반환만)
- 토스트, 타이머, 모달 열기 등 사이드이펙트는 반드시 업데이터 밖에서 실행
- 단일 토글 방식에서는 버그가 안 보여도, 누적 방식으로 바뀌면 드러남 — 처음부터 분리하는 게 안전
